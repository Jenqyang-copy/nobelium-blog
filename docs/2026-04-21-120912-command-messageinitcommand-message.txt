  二、可继续个性化的维度

  1. 内容与 Notion

  - NOTION_PAGE_ID（Vercel 环境变量）：Notion 数据库 ID，必填。
  - NOTION_ACCESS_TOKEN：仅私有库需要；每 180 天过期，图片渲染会受限。
  - 发布过滤逻辑：lib/notion/filterPublishedPosts.js
  控制什么算"已发布"（status、type、date），可在此修改收录规则。
  - sortByDate: true 可开启按日期排序（当前 false，顺序跟随 Notion 视图）。

  2. 导航与站点信息

  - showAbout：开启后需要在 Notion 里创建 type=Page 且 slug=about 的页面。
  - showArchive：启用归档页。
  - autoCollapsedNavBar：顶栏交互风格。
  - 导航项定义在 components/Header.js 的 NavBar（添加项在这里改）。
  - 默认 favicon：public/favicon.png + public/favicon.dark.png（暗色自动切换，见
   Header.js:41）。

  3. 视觉

  - 背景色：lightBackground / darkBackground → 自动透传 Tailwind 的 bg-day /
  bg-night。
  - 字体：改 font 字段切换 sans/serif；具体字体栈在 consts.js。本地字体文件在
  public/fonts/（IBM Plex Sans + Source Serif，vercel.json
  配置了一年不可变缓存）。
  - 全局样式：styles/globals.css、styles/notion.css（Notion 区块覆盖样式）。
  - Tailwind：tailwind.config.js（darkMode: 'class'）。
  - 站点
  logo/截图：Nobelium-Logo.svg、desktop.png、pageid.png（可删除，仓库演示用）。

  4. SEO / OG

  - seo.keywords：关键词数组。
  - seo.googleSiteVerification：填入可验证 Search Console。
  - ogImageGenerateURL：可部署自己的 og-image 替换默认的 craigary
  服务，避免依赖三方。
  - Sitemap：next-sitemap.config.js 在 postbuild 自动生成 robots.txt +
  sitemap.xml，读取 config.link。
  - next.config.js 加了 Permissions-Policy: interest-cohort=() 禁用 FLoC。

  5. 分析（目前未启用自建方案）

  - analytics.provider = 'ga'：填
  analytics.gaConfig.measurementId（G-XXXXXXXXXX）。
  - analytics.provider = 'ackee'：填 tracker / dataAckeeServer / domainId。
  - Vercel Analytics：已通过 @vercel/analytics
  始终启用（pages/_app.js），无需配置，但仅在 Vercel 平台生效。

  6. 评论（目前未启用）

  - comment.provider = 'gitalk'：配 repo/owner/admin/clientID/clientSecret。
  - comment.provider = 'utterances'：仅需 repo（GitHub App 授权）。
  - comment.provider = 'cusdis'：配 appId（自建可改 host / scriptSrc）。

  7. 部署

  - vercel.json：cleanUrls: true、字体长缓存。
  - Dockerfile：可自建 docker 镜像运行。
  - Node 版本固定 18.x（见 package.json engines）。

  三、建议的后续个性化动作（按优先级）

  1. SEO 加固：填 seo.googleSiteVerification；把 seo.keywords
  改成更贴合你身份的词；考虑自建 og-image 服务替换 ogImageGenerateURL。
  2. 启用评论：推荐 utterances（最简单，只要一个 GitHub repo）。
  3. 站点图标：把 public/favicon.png / favicon.dark.png 换成你的品牌。
  4. 开启按日期排序：sortByDate: true，否则顺序依赖 Notion 视图手工拖动。
  5. 分析：如果想脱离 Vercel，加一个 ga 或 ackee；否则当前 @vercel/analytics
  已够用。
  6. .github/ 里还有模板/工作流，可按需精简（当前是上游仓库的）。
